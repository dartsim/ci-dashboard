# DART CI Dashboard

Static dashboard for the `dartsim/dart` GitHub Actions queue and runner pools.

The dashboard is generated by a scheduled GitHub Actions workflow and served
from the `gh-pages` branch by GitHub Pages. It keeps GitHub API credentials
server-side and publishes only sanitized JSON and static assets.

## Published Site

Expected public URL:

https://dartsim.github.io/ci-status/

The generated data is also published to this repository's `gh-pages` branch so
the website can load it without committing generated dashboard data back to the
website repository on every refresh.

## What It Shows

- queued and in-progress jobs grouped by runner label
- exact self-hosted runner status when the token can read runner metadata
- inferred GitHub-hosted runner pressure from queued jobs and queue wait
- recent queue latency, completion, cancellation, and failure statistics
- recent CUDA/GPU job status
- warnings when jobs are queued despite an idle matching self-hosted runner

GitHub does not expose exact GitHub-hosted runner capacity through the public
Actions APIs, so official runner availability is inferred from queue behavior.

## Required Secret

For full self-hosted runner visibility, add a repository secret named
`DASHBOARD_GITHUB_TOKEN` to this repository.

Recommended fine-grained personal access token permissions:

- Repository: `dartsim/dart`
- Metadata: read
- Actions: read
- Administration: read

The scheduled workflow requires this secret before it publishes, so it does not
accidentally overwrite the public site with degraded runner data.

## Local Development

```bash
cd /home/js/dev/dartsim/ci-dashboard
GITHUB_TOKEN="$(gh auth token)" python3 scripts/collect.py
python3 -m http.server 8000 --directory public
```

Then open http://localhost:8000/.
